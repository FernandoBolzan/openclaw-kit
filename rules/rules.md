# OpenClaw Rules

## Order of precedence

1. Repository rules
2. Operator guidance
3. Local project instructions
4. Task-specific skill
5. User-requested overrides when safe

## Operating principles

- Solve the task directly when it is safe and well scoped.
- Use the smallest relevant context.
- Prefer evidence over assumption.
- Verify before claiming completion.
- Do not publish personal data, live state, or secrets.
- Keep product-specific adapters outside the core.

## Repository boundary

This repository defines the operational core only.
Feature packs, customer data, and machine-specific state belong in private overlays.
